HTTPS

cookie

储存在客户端的一小块信息

session

session储存在服务端，sessionid储存在客户端cookie中，通过与服务端的session匹配登录（如果失效了就不再能登录），极大增大了服务器的负担

token

token储存在客户端，发送给服务端解密放到header检测得到access token
但是access token容易失效，所以有refresh token去与服务端交流得到新的refresh token和access token。

但是，token不利于记录对话，是无状态的，记录对话用session
